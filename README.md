# Integration_gateway
Интеграционный сервис для обеспечения автоматизированной трансформации и передачи данных между [Система 1] (XML) и [Система 2] (JSON). <br>
## Цель проекта: <br>
Автоматизация процесса обмена данными для исключения ручного ввода сотрудниками НСИ, сокращение времени передачи с нескольких часов до секунд и минимизация рисков, связанных с человеческим фактором.<br>
## Ключевые возможности Mapping Engine: <br>
* Реализация гибкой трансформации данных XML ↔ JSON. <br>
* Управление правилами трансформации на уровне БД с использованием статусной модели (CREATED, APPROVAL, ENABLED, DISABLED, DELETED). <br>
* Отказоустойчивость: Внедрение Retry Policy с экспоненциальной задержкой (1с, 2с, 4с, 8с) и механизм транзакционного отката (Rollback) для обеспечения атомарности операций. <br>
* Валидация: Строгая проверка входящих XML-структур на соответствие XSD-схемам. <br>
## Архитектурные решения  <br>
* Стек: REST API, PostgreSQL (JSONB для хранения правил трансформации). <br>
* Модель данных: Связь 1:1 между registry_rules и rule_storage для оптимизации производительности запросов метаданных. <br>
* Масштабируемость: Архитектура позволяет добавлять новые правила трансформации без изменения ядра сервиса. <br>
